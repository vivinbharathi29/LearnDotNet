Excalibur Help - Deliverable Releases

Definitions

Before you can release your deliverables with Excalibur, there are a few terms you need to understand:

  • Root Deliverable: Instead of making you enter a lot of information every time you need to release a new version of a deliverable, Excalibur collects all of the information that is the same or almost always the same across all versions in the Root Deliverable.  You can think of the Root Deliverable as the place you enter all of the default values for new versions.  Since this information is entered only once, the process of entering all versions (other than the first one) should be greatly simplified. 
  • Deliverable Version: The deliverable version is a collection of all of the information about a particular version (version, revision, and pass) of a deliverable that you are releasing. This includes things like which languages, products, and operating systems are supported by the release, vendor version number, Observations fixed and other changes made, developer's name, etc. 
  • HFCN:  [Note: The HFCN tool should no longer be used for new releases] - HFCN is a legacy tool that was being used mainly in Consumer Desktops.  It provided release notices for hardware and firmware releases.  The functionality of that tool has been imported into Excalibur to allow developers to release new versions of hardware and firmware.  

    Step 1: Start Excalibur

    1. Start Excalibur  [Tell Me More] 

    Step 2: Enter a Root Deliverable

    Note: If your Root Deliverable has already been entered into Excalibur, skip to Step 3: Enter a Deliverable Version . If you are not sure if your Root Deliverable has been entered yet, you can Browse the Root Deliverable List.

    1. Click "Add New" link in the Root Deliverable section of the screen
    2. Fill in all of the information that applies to your deliverable. 
      1. Items with a red asterisks (*) are required.
      2. Since these screens support many different types of deliverables, you will probably not need to enter information into all of the fields.  In general, if you don't know what a field is used for, you probably don't need to use it for your type of deliverable.
    3. On the last screen of the wizard, you are asked if you would like to add this deliverable to your Favorites List.  The Favorites Lists is a list of deliverables or products that you work with a lot.  It is located in the left pane of the main screen.   This list allows you quick access to your deliverables and speeds up the process of releasing deliverable versions. 

    Step 3: Enter a Deliverable Version

    1. Select the appropriate Root Deliverable for the version you are entering.  There are two methods of selecting a Root Deliverable
      1. Click the Root Deliverable in your Favorites List (if you previously added it) OR
      2. Click the "Find" link in the Root Deliverable section of the main screen.  Then select your root deliverable from the list of all Root Deliverables.  Once you have a Root Deliverable Displayed, you can click the "Add to Favorites" link if you want to add it to your Favorites list.
    2. Click the "Add New Version" link on the Root Deliverable page.  This will start a wizard that pre-populates the version with all of the defaults you entered when you created the Root Deliverable.
    3. Fill in all of the fields in the wizard that apply to your version.
      1. Items with a red asterisks (*) are required.
      2. Since these screens support many different types of deliverables, you will probably not need to enter information into all of the fields.  In general, if you don't know what a field is used for, you probably don't need to use it for your type of deliverable.
    4. On the last screen in the wizard, you will get an opportunity to review all of the selections you made in the wizard.  If everything looks right, click Finish to release the version and notify everyone you specified in the "Send Email To" field.
    5. You will be returned to the deliverable root screen and the new version you just entered will have been added to the top of the Versions list.
    Deliverable Properties
    1. If you need to view or modify the properties of a Root Deliverable, you can click the "Properties" link the Root Deliverable page.
    2. If you need to view or modify the properties of a Deliverable Version, you can click the deliverable version in the Version list on a Root Deliverable screen and choose "Properties" from the popup menu.
